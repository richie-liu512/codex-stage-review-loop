# Codex Stage Review Loop

这是一个轻量级 Codex Skill，用于在完成一个任务片段、小项目改动或 Skill 编辑后，快速做自检、审查和必要修复。

它的目标不是承诺“完美”，而是让 Codex 在当前范围内确认：需求是否落实、验证是否通过、有没有明显 Bug、有没有需要用户确认的关键决策。

## 功能

- 按用户原始要求做自我验收。
- 用新视角重新审查，尽量发现遗漏、逻辑漏洞和体验问题。
- 支持可选的独立审查者模式，用于更接近盲审的检查。
- 只自动修复已验证的阻断问题。
- 把可选重构、风格优化、额外功能列为建议，不擅自扩大范围。
- 遇到需求歧义、风险操作或产品取舍时，停止并询问用户。

## 两种模式

### 轻量模式

默认模式，适合日常频繁使用。最多执行两轮检查和修复。

适合：

- 做完一个小功能后快速检查
- 编辑完一个 Skill 后验收
- 继续下一步前排除明显问题
- 不想启动重型最终审查流程

示例：

```text
使用 $stage-review-loop 轻量检查刚才的改动，并修复真实问题。
```

### 独立审查者模式

更强的审查模式。可用时优先使用独立 reviewer 或新的审查视角；不可用时，会用受限上下文模拟新视角审查。

适合：

- 发布前确认
- 审查风险较高的改动
- 检查最终结果是否真正符合原始需求

示例：

```text
使用 $stage-review-loop 独立审查者模式盲审这个 Skill。
```

## 安装

把 Skill 文件夹复制到 Codex skills 目录：

```text
stage-review-loop/
  SKILL.md
  agents/
    openai.yaml
```

然后按名称调用：

```text
使用 $stage-review-loop 检查当前工作。
```

## 设计原则

这个 Skill 刻意保持轻量：

- 有循环上限，避免无限纠结
- 只修复有证据的阻断问题
- 不把可选建议变成擅自改动
- 遇到风险和歧义先问用户
- 不声称结果“完美”
